# 🚀 Docker 下载加速指南 (Mac 专用)

如果你在拉取 `ghcr.io/openclaw/openclaw` 镜像时进度条不动或者只有几 KB/s，请尝试以下方案：

## 方案一：为 Docker Desktop 配置代理 (最推荐)

这是针对 Mac 用户解决 GitHub/GHCR 访问慢最稳定、最标准的方法。

1.  打开 **Docker Desktop**。
2.  点击右上角的 **齿轮图标 (Settings)**。
3.  在左侧导航栏选择 **Resources** -> **Proxies**。
4.  开启 **Manual proxy configuration**。
5.  在 `Web Server (HTTP)` 和 `Secure Web Server (HTTPS)` 中填入你的本地代理地址（通常如果你开了“加速工具”，默认是 `http://127.0.0.1:7890` 或类似端口）。
6.  点击 **Apply & Restart**。
7.  **重启后**，回到终端再次运行：
    ```bash
    docker compose up -d
    ```

## 方案二：使用国内镜像加速器 (针对 Docker Hub)

> **注意**：GHCR 是 GitHub 自己的，国内很多纯镜像站可能没同步。但你可以尝试配置以加速基础层下载。

1.  在 Docker Desktop 设置中选择 **Docker Engine**。
2.  在右侧的 JSON 配置中增加 `"registry-mirrors"`，例如：
    ```json
    {
      "registry-mirrors": [
        "https://docker.m.daocloud.io",
        "https://dockerproxy.com"
      ]
    }
    ```
3.  点击 **Apply & Restart**。

## 方案三：使用 docker-pull 脚本代理

如果你不想全局改 Docker 设置，可以尝试在拉取命令前临时注入代理环境变量（虽然 Docker Engine 有时会忽略，但在 Mac 上直接配置 Settings 的效果最好）。

## 方案四：耐心与断点续传

Docker 的拉取是分块的，只要有一两个块跑完了，下次就会快很多。
- 如果卡死很久没动，可以 `Ctrl + C` 杀掉，然后**立即重新运行** `docker compose up -d`。
- 它会跳过已完成的部分，只下载剩下的部分。

---
**提示**：一旦这几百兆下载完，你的 OpenClaw 就永久保存在本地了，以后启动**完全不需要流量**。
